## 1. 产品概述

### 1.1 背景
现有 Events & Rumors 系统提供了基础的事件管理和谣言传播功能。本扩展旨在增强系统能力，支持批量操作、可视化分析和数据导出。

### 1.2 目标
- 提升 DM 工作效率：支持批量导入历史事件
- 增强玩家体验：可视化展示谣言传播路径
- 支持数据分析：导出事件时间线用于外部分析

### 1.3 用户故事细化

#### US-001: 批量导入事件 (P0)
**作为** DM  
**我希望** 通过 CSV/JSON 文件批量导入历史事件  
**以便** 快速初始化世界历史，减少重复性手工录入工作

**验收标准**:
- 支持 CSV 格式（标准表格结构）
- 支持 JSON 格式（嵌套对象结构）
- 单次最多导入 1000 个事件
- 提供详细的导入报告（成功/失败统计）
- 失败的记录需要提供错误原因
- 导入过程是原子性的（全部成功或全部失败）

#### US-002: 谣言传播可视化 (P1)
**作为** 玩家  
**我希望** 查看谣言传播的可视化图表  
**以便** 理解信息在世界中的流动路径

**验收标准**:
- 以图数据结构返回（节点+边）
- 节点包括：地点、谣言
- 边包括：传播路径（带时间戳）
- 支持按谣言 ID 过滤
- 支持按时间范围过滤
- 节点包含元数据（位置坐标、真实度等）

#### US-003: 事件时间线导出 (P2)
**作为** 分析师  
**我希望** 导出事件时间线为 PDF/PNG 格式  
**以便** 进行离线分析和演示

**验收标准**:
- 支持 PDF 导出（分页文档）
- 支持 PNG 导出（图片格式）
- 按时间顺序排列事件
- 包含事件详情（名称、描述、日期）
- 支持自定义时间范围
- 支持按事件类型过滤

## 3. API 设计

### 3.1 批量导入 API

**POST** `/api/world/{world_id}/events/bulk-import`

请求体：
- format: csv | json
- data: base64 编码的文件内容
- options: 导入选项（原子性、验证、生成谣言）

响应：
- success: 是否成功
- total/imported/failed: 统计信息
- imported_ids: 导入的事件 ID 列表
- errors: 错误详情列表

### 3.2 传播可视化 API

**GET** `/api/world/{world_id}/rumors/visualization`

查询参数：
- rumor_id: 可选，特定谣言 ID
- from_date/to_date: 可选，时间范围
- max_hops: 可选，最大传播跳数

响应：
- graph.nodes: 节点列表（地点、谣言）
- graph.edges: 边列表（传播路径）
- metadata: 统计信息

### 3.3 时间线导出 API

**GET** `/api/world/{world_id}/events/export`

查询参数：
- format: pdf | png | json
- from_date/to_date: 可选，时间范围
- event_types: 可选，事件类型过滤

响应：
- JSON: 时间线数据结构
- PDF/PNG: 二进制文件流

## 4. 技术实现

### 4.1 批量导入流程

1. 接收请求 (Router)
2. 解码 base64 数据 (Service)
3. 解析文件 (CSV/JSON Parser)
4. 验证每行数据 (Domain Validator)
5. 开始事务
6. 创建 Event 实体（批量）
7. 保存到 Repository
8. 可选：生成 Rumors
9. 提交/回滚事务
10. 返回导入报告

### 4.2 可视化数据构建

1. 获取所有 Rumors
2. 应用过滤条件
3. 构建图结构（节点+边）
4. 返回 Graph JSON

### 4.3 导出流程

1. 获取过滤后的事件列表
2. 按日期排序
3. 构建时间线数据结构
4. 根据格式生成输出

| CSV 编码问题 | 中 | 支持 UTF-8/BOM |
| 复杂图性能 | 中 | 限制最大节点数 |
