# Platform Middleware

This document defines the backend middleware contract for the canonical FastAPI app in `src/apps/api/main.py`.

## Order

Middleware is installed in this order:

1. `LoggingMiddleware`
2. `MetricsMiddleware`
3. `CorrelationIdMiddleware`

Starlette wraps middleware in reverse order at runtime, so correlation id extraction is available to downstream middleware and route handlers through `request.state.correlation_id`.

## Correlation And Request IDs

- `X-Correlation-ID` is accepted from the client when present.
- `X-Request-ID` is accepted from the client when present.
- Missing ids are generated by `CorrelationIdMiddleware`.
- Both ids are written to `request.state`.
- Both ids are returned on the HTTP response.
- `get_correlation_id(request)` and `get_request_id(request)` are the canonical helpers for middleware and handlers that need the ids.

The legacy import path `src.apps.api.middleware.logging.LoggingMiddleware` is a compatibility export. New code should import `LoggingMiddleware` from `src.shared.infrastructure.middleware`.

## Error Response Contract

The API error handler preserves the public JSON shape:

```json
{
  "error": {
    "code": "ERROR_CODE",
    "message": "Human-readable message",
    "details": {}
  }
}
```

Unhandled exceptions return `INTERNAL_SERVER_ERROR` without leaking implementation details. Validation errors return structured details for the failing fields.

## Metrics

`MetricsMiddleware` records:

- request count by method, path, and status code
- request duration by method and path
- in-progress request gauge by method and path

The in-progress gauge must decrement in success and failure paths. Exceptions are re-raised after metrics are recorded so the error handler can preserve the HTTP contract.

## CORS

The default CORS behavior remains centralized in `src.apps.api.middleware.cors.get_cors_config`. Production configuration filters wildcard origins unless wildcard mode is explicitly selected by environment configuration.

## Coverage Gate

Run the fast middleware and circuit breaker coverage gate before changing cross-cutting infrastructure:

```bash
uv run pytest \
  tests/shared/infrastructure/circuit_breaker \
  tests/shared/infrastructure/middleware \
  tests/apps/api/middleware \
  --cov=src/shared/infrastructure/circuit_breaker \
  --cov=src/shared/infrastructure/middleware \
  --cov=src/apps/api/middleware \
  --cov-report=term-missing \
  --cov-fail-under=80 \
  -q
```
