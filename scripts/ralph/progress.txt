# Ralph Progress - Operation Vanguard (Pre-Simulation Prep)

## Codebase Patterns

### Backend Architecture
- Use `from src.core.result import Ok, Err, Result` for Result pattern
- Value objects use `@dataclass(frozen=True)` with `__post_init__` for validation
- Hexagonal architecture: Routers -> Services -> Repositories -> Domain
- Domain layer MUST NOT import from API layer (enforced by import-linter)
- Import path pattern: `from src.contexts.<context>.domain.value_objects.<name> import <Class>`
- Two app factories exist: src.api.main_api_server:create_app and src.api.app:create_app
- Error responses use ErrorDetail schema with 'code', 'message', 'details' fields

### Frontend Architecture
- Frontend components use shadcn/ui from @/shared/components/ui/ (Card, Button, etc.)
- Frontend TanStack Query hooks: use queryClient.invalidateQueries for cache invalidation
- Frontend API hooks are in frontend/src/lib/api/*.ts
- shadcn/ui components ONLY - no custom CSS classes
- Component files should be <300 lines

### Testing
- Test files need `pytestmark = pytest.mark.unit` at module level
- Mock `aioredis` in tests: `sys.modules["aioredis"] = MagicMock()`
- E2E tests import Page type from '@playwright/test' at top of file
- pytest markers required: @pytest.mark.unit, @pytest.mark.integration, @pytest.mark.e2e

### Schema Split Investigation Findings
- 95 schemas across 10 domains, 3,704 lines total
- Character: 7 schemas, 398 lines
- Narrative Streaming: 22 schemas, 1,219 lines (most complex)
- Smart Tagging: 8 schemas, 1,124 lines
- World Simulation: 21 schemas, 231 lines
- Minimal cross-dependencies between schemas

### Domain Violation Investigation Findings
- Only 4 violations found (not 11 as estimated)
- All are schema imports from API layer
- Files: llm_character_generator.py, scene_generator_port.py, scene_service.py, llm_scene_generator.py
- Fix: Create domain value objects, use dependency inversion

### Frontend Decomposition Investigation Findings
- BrainSettingsPage.tsx: 2,109 lines, 8 logical sections
- Recommended split: 5 components (not 3)
  - ModelConfigPanel (~400 lines)
  - TokenAnalyticsPanel (~500 lines)
  - RealtimeUsagePanel (~120 lines)
  - ApiKeyPanel (~250 lines)
  - RagConfigPanel (~190 lines)
- Shared state: config, stats, brainSettings, isSaving, date filters
