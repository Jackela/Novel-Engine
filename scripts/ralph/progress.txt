# Ralph Progress Log

## Codebase Patterns
- Use `from src.core.result import Ok, Err, Result` for Result pattern
- Value objects use `@dataclass(frozen=True)` with `__post_init__` for validation
- Test files need `pytestmark = pytest.mark.unit` at module level
- Mock `aioredis` in tests: `sys.modules["aioredis"] = MagicMock()`
- Import path pattern: `from src.contexts.<context>.domain.value_objects.<name> import <Class>`
- Domain events can accept Union types for backward compatibility (datetime | str)
- from_dict class methods should handle backward compatibility for renamed fields
- Two app factories exist: src.api.main_api_server:create_app and src.api.app:create_app - routers must be registered in both
- Tests use `import api_server` which loads src.api.app, not src.api.main_api_server
- Error responses use ErrorDetail schema with 'code', 'message', 'details' fields (not 'detail')
- Frontend TanStack Query hooks pattern: use queryClient.invalidateQueries for cache invalidation, onMutate/onError/onSettled for optimistic updates
- Frontend API hooks are in frontend/src/lib/api/*.ts, feature-specific hooks can go in frontend/src/features/*/api/
- Frontend components use shadcn/ui from @/shared/components/ui/ (Card, Button, etc.)
- E2E tests import Page type from '@playwright/test' at top of file, not inline
