## Codebase Patterns
- **Architecture**: We follow strictly Hexagonal Architecture (Ports & Adapters).
- **Contract-First**: Always define Pydantic models in `src/api/schemas.py` BEFORE writing logic.
- **Frontend**: Frontend types must be synced from OpenAPI via `npm run api:sync` (or equivalent command).
- **Testing**: Use `pytest` for backend and `npm run test` for frontend.
- **Domain Value Objects**: Place in `src/contexts/{domain}/domain/value_objects/` with validation in `__post_init__`.
- **Ports**: Define interfaces in `src/contexts/{domain}/application/ports/` using Protocol.
- **Type Aliases**: Use aliases for backwards compatibility when refactoring (e.g., `CharacterGenerationResult = CharacterCard`).
