# Proposal: Historical Events and Rumor Propagation

## Problem
The world has time and faction intents, but it lacks a persistent chronological history of "what actually happened" and a mechanism for how ordinary people learn about these happenings (information delay).

## Proposed Solution
1. **Historical Events**: Leverage the existing `HistoryEvent` entity to record major occurrences, either auto-generated by the Simulation Engine resolving Faction Intents, or manually authored by users.
2. **Rumor System**: Use the existing `Rumor` entity and `RumorPropagationService`. When an event occurs, it spawns `Rumor`s with truth values perturbed by ±10%. These rumors spread to adjacent locations on each time tick.

## Key Design Decisions

### 1. Event Source: Both Manual and Auto-Generated (Option C)
- **Manual**: Authors can seed world history (e.g., "Dragons went extinct 10,000 years ago")
- **Auto-generated**: Simulation creates events when faction intents resolve (wars, alliances, etc.)

### 2. Truth Value Perturbation
- Base truth by impact: GLOBAL 90%, REGIONAL 70%, LOCAL 50%
- Add random perturbation: ±10% for organic feel
- Result range: 80-100% (GLOBAL), 60-80% (REGIONAL), 40-60% (LOCAL)

### 3. Propagation Rate: Simple Tick-Based (Option A)
- O(n) complexity for performance
- Each tick: rumors spread to adjacent locations via `find_adjacent()`
- Complex pathfinding deferred to future iterations

## Scope
- **Domain**: Use existing `HistoryEvent` and `Rumor` entities
- **Application**: Create `RumorPropagationHandler` for EventBus integration
- **API/Frontend**: World Timeline visualizer and Tavern Rumor Board

## Success Criteria
- [ ] Rumors propagate automatically when time advances
- [ ] Manual event creation works via API
- [ ] WorldTimeline displays events chronologically
- [ ] TavernBoard shows location-specific rumors with truth indicators
